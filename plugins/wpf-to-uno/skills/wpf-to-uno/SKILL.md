# WPF ŌåÆ Uno Platform Migration Skill

Uno Platform is a cross-platform UI framework that targets Windows (WinUI 3), macOS, Linux, iOS,
Android, and WebAssembly from a single codebase. It uses WinUI 3 / UWP-style XAML ŌĆö **not** WPF
XAML ŌĆö so migration requires understanding the dialect shift as well as architectural differences.

> For deep dives on any section, load the matching reference file listed at the end of this skill.

---

## 1. Project Structure & Setup

**WPF project** ŌåÆ `.csproj` targeting `net9.0-windows`, single platform.

**Uno Platform project** ŌåÆ Multi-head solution generated by `dotnet new unoapp` or the Uno Platform
Template Wizard. Key heads:
- `MyApp` ŌĆö shared library (XAML + ViewModels + business logic)
- `MyApp.Windows` ŌĆö WinUI 3 desktop head
- `MyApp.Mobile` ŌĆö iOS / Android
- `MyApp.Skia.Desktop` ŌĆö Linux / macOS (Skia rendering)
- `MyApp.Wasm` ŌĆö WebAssembly

**Migration starting point**: Keep business logic in the shared project. Only platform-specific code
goes into head projects using `#if` directives or partial classes.

## 2. XAML Dialect Differences

WPF uses its own XAML dialect; Uno uses **WinUI 3 / UWP XAML**. These are related but distinct.

### Namespace changes

| WPF | Uno Platform (WinUI 3) |
|-----|------------------------|
| `xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"` | `xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"` Ō£ģ same root |
| `xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"` | `xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"` Ō£ģ same |
| `xmlns:d="http://schemas.microsoft.com/expression/blend/2008"` | Same Ō£ģ |
| `xmlns:mc="http://schemas.openxmlformats.org/markup-compatibility/2006"` | Same Ō£ģ |
| `xmlns:local="clr-namespace:MyApp"` | `xmlns:local="using:MyApp"` ŌÜĀ’ĖÅ **`using:` not `clr-namespace:`** |

### Key XAML property / syntax differences

| WPF | Uno / WinUI 3 | Notes |
|-----|---------------|-------|
| `x:Static` | Not supported | Use `x:Bind` or converters |
| `{Binding RelativeSource={RelativeSource Self}}` | `x:Bind` preferred | RelativeSource is limited |
| `Trigger`, `DataTrigger`, `EventTrigger` | **Not supported** | Use `VisualStateManager` |
| `Style.Triggers` | **Not supported** | Use VSM or `x:Bind` |
| `MultiBinding` | Limited / use converters | `x:Bind` with functions preferred |
| `StringFormat` in Binding | Not supported | Use converters or `x:Bind` |
| `IValueConverter.Convert` parameter `culture` | Passed as null on some platforms | Don't rely on it |
| `DockPanel` | Not built-in | Use `Grid` or install WCT |
| `WrapPanel` | Not built-in | Use `ItemsWrapGrid` inside a `ScrollViewer` |
| `UniformGrid` | Not built-in | Replicate with `Grid` column/row definitions |
| `GroupBox` | Not built-in | Use `Border` + `TextBlock` header pattern |
| `Expander` | Available via WCT | `Microsoft.UI.Xaml.Controls.Expander` (WinUI) |
| `DataGrid` | Available via WCT | See Section 4 |
| `FlowDocument` / `RichTextBox` | Not supported | Use third-party or `WebView2` |

### x:Bind vs Binding

Prefer `x:Bind` (compiled bindings) over `{Binding}` in Uno for performance and correctness:
```xml
<!-- WPF -->
<TextBlock Text="{Binding UserName}" />

<!-- Uno preferred -->
<TextBlock Text="{x:Bind ViewModel.UserName, Mode=OneWay}" />
```

`x:Bind` requires a typed `DataContext` ŌĆö set `x:DataType` on the root or the binding target.

## 5. Custom Controls Migration

### WPF CustomControl ŌåÆ Uno CustomControl

The pattern is similar but with WinUI 3 conventions:

```csharp
// WPF
public class MyControl : Control {
    static MyControl() {
        DefaultStyleKeyProperty.OverrideMetadata(typeof(MyControl),
            new FrameworkPropertyMetadata(typeof(MyControl)));
    }
}

// Uno / WinUI 3
public partial class MyControl : Control {
    public MyControl() {
        this.DefaultStyleKey = typeof(MyControl);
    }
}
```

Place the default style in `Themes/Generic.xaml` ŌĆö same as WPF.

### Key differences

| Aspect | WPF | Uno / WinUI 3 |
|--------|-----|---------------|
| DependencyProperty registration | `DependencyProperty.Register(...)` Ō£ģ same | Same API |
| PropertyChangedCallback | `FrameworkPropertyMetadata` | `PropertyMetadata` (no `FrameworkPropertyMetadata`) |
| `AffectsRender` / `AffectsMeasure` flags | Via `FrameworkPropertyMetadataOptions` | Not available; call `InvalidateArrange()` manually |
| `OnRender(DrawingContext)` | WPF-specific | Override `ArrangeOverride` / use `ShapeBase` / SkiaSharp |
| Custom drawing | `DrawingContext` via `OnRender` | **SkiaSharp** via `SKXamlCanvas` (cross-platform) |
| `Visual` tree manipulation | WPF VisualTreeHelper | `VisualTreeHelper` Ō£ģ available |
| `TemplateBinding` | Ō£ģ | Ō£ģ same |
| `GetTemplateChild` | Ō£ģ | Ō£ģ same |

### SkiaSharp for custom rendering

For controls requiring low-level drawing (charts, gauges, canvas apps):
```xml
<skia:SKXamlCanvas xmlns:skia="using:SkiaSharp.Views.Windows"
                   PaintSurface="OnPaintSurface" />
```
```csharp
void OnPaintSurface(object sender, SKPaintSurfaceEventArgs e) {
    var canvas = e.Surface.Canvas;
    canvas.Clear(SKColors.Transparent);
    // SkiaSharp drawing calls
}
```

## 6. C# Code-Behind Differences

### Lifecycle events

| WPF | Uno / WinUI 3 |
|-----|---------------|
| `Window.Loaded` | `Page.Loaded` |
| `Window.Closing` | `Window.Closed` (no cancel in WinUI) |
| `Application.Startup` | `App.OnLaunched(LaunchActivatedEventArgs)` |
| `Application.Exit` | `App.OnWindowClosed` / platform lifecycle |
| `Dispatcher.Invoke` | `DispatcherQueue.TryEnqueue(...)` |
| `Application.Current.Dispatcher` | `Microsoft.UI.Dispatching.DispatcherQueue.GetForCurrentThread()` |

### Threading

```csharp
// WPF
Application.Current.Dispatcher.Invoke(() => { /* UI update */ });

// Uno / WinUI 3
_dispatcherQueue.TryEnqueue(() => { /* UI update */ });
// Or in a control:
DispatcherQueue.TryEnqueue(DispatcherQueuePriority.Normal, () => { /* UI update */ });
```

### Common API replacements

| WPF API | Uno / WinUI 3 Equivalent |
|---------|--------------------------|
| `MessageBox.Show(...)` | `ContentDialog` (async) |
| `OpenFileDialog` | `FileOpenPicker` |
| `SaveFileDialog` | `FileSavePicker` |
| `FolderBrowserDialog` | `FolderPicker` |
| `System.Windows.Clipboard` | `Windows.ApplicationModel.DataTransfer.Clipboard` |
| `System.Windows.Application.GetResourceStream` | `StorageFile` / embedded resources |
| `System.Drawing.*` | `Microsoft.UI.Xaml.Media.*` or SkiaSharp |
| `Color` (System.Windows.Media) | `Windows.UI.Color` |
| `Brush` (System.Windows.Media) | `Microsoft.UI.Xaml.Media.Brush` |
| `Point`, `Size`, `Rect` | `Windows.Foundation.Point/Size/Rect` |

## 7. Navigation Patterns

### WPF patterns (frame-based or window-based)

WPF typically uses:
- Multiple `Window` instances
- `NavigationWindow` + `Page`
- Manual `ContentControl` swapping
- Prism `RegionManager`

### Uno Platform navigation

Uno recommends the **Uno.Extensions Navigation** library or manual `Frame` navigation.

#### Option A: Frame-based navigation (simple apps)

```csharp
// In App.xaml.cs or Shell
var frame = new Frame();
frame.Navigate(typeof(HomePage));
Window.Content = frame;

// Navigate forward
frame.Navigate(typeof(DetailPage), parameter);

// Navigate back
if (frame.CanGoBack) frame.GoBack();
```

#### Option B: Uno.Extensions Navigation (recommended for complex apps)

```csharp
// Registration
services.AddNavigation(nav => nav
    .Register<HomePage>("Home")
    .Register<DetailPage>("Detail")
);

// Navigate
await _navigator.NavigateRouteAsync(this, "Detail", data: myItem);
```

#### Option C: ReactiveUI (if using ReactiveUI MVVM)

```csharp
// RoutingState drives navigation
Router.Navigate.Execute(new DetailViewModel(item));
```

### Window management

WPF apps often open multiple `Window` instances. In Uno:
- **Windows (WinUI 3)**: Multiple windows are supported via `AppWindow`
- **Mobile/WASM**: Single window only ŌĆö redesign multi-window flows as navigation or dialogs
- Use `ContentDialog`, `Flyout`, or `Popup` to replace secondary windows on constrained platforms

```csharp
// Conditional multi-window (Windows only)
#if WINDOWS
    var newWindow = new Window();
    newWindow.Content = new MyView();
    newWindow.Activate();
#else
    // Navigate or show dialog instead
#endif
```

## 8. MVVM Libraries

### Supported on Uno Platform

| Library | NuGet | Uno Support | Notes |
|---------|-------|-------------|-------|
| **CommunityToolkit.Mvvm** | `CommunityToolkit.Mvvm` | Ō£ģ Full | Recommended; source-gen `[ObservableProperty]`, `[RelayCommand]` |
| **ReactiveUI** | `ReactiveUI` + `ReactiveUI.Uno` | Ō£ģ Full | Powerful; steeper learning curve |
| **Prism** | `Prism.Uno` | Ō£ģ Community port | Navigation + DI included |
| **MVVM Light** | (deprecated) | ŌÜĀ’ĖÅ Avoid | Superseded by CommunityToolkit.Mvvm |
| **Caliburn.Micro** | `Caliburn.Micro` | ŌÜĀ’ĖÅ Partial | Convention-based; limited Uno support |

### Recommended: CommunityToolkit.Mvvm

```csharp

public partial class MainViewModel : ObservableObject {
    [ObservableProperty]
    private string _userName = string.Empty;

    [RelayCommand]
    private async Task LoadDataAsync() { /* ... */ }
}
```

### WPF Prism ŌåÆ Uno Prism

If migrating a Prism WPF app:
- Replace `PrismApplication` (WPF) with `PrismApplication` from `Prism.Uno`
- Replace `IRegionManager` region-based navigation with `Frame`-based or Uno.Extensions navigation
- `RegionManager.RequestNavigate` ŌåÆ `_navigator.NavigateRouteAsync(...)`
- Module loading works but XAML module discovery differs; use explicit registration

## 9. Deprecated Technologies (Do Not Port)

These WPF/Windows technologies have **no Uno equivalent** and must be redesigned:

| Technology | Action |
|------------|--------|
| `WindowsFormsHost` | Remove; replace embedded WinForms controls with native equivalents |
| `WebBrowser` (WPF) | Replace with `WebView2` (Windows only) or `WebView` |
| `AxHost` (ActiveX) | **Not supported anywhere** ŌĆö replace with modern alternatives |
| `RichTextBox` (full WPF) | Use `WebView2` with HTML editor, or a commercial control |
| `FlowDocument` / `XPS` | Render via `WebView2` or generate PDF with a library |
| `XmlDataProvider` | Replace with code-behind data loading |
| `ObjectDataProvider` | Replace with ViewModel data binding |
| `BitmapEffect` (old WPF) | Already deprecated in WPF; use `Effect` or SkiaSharp |
| `3D (Viewport3D / Visual3D)` | Replace with SkiaSharp 3D, BabylonJS (WASM), or Unity |
| `System.AddIn` (MAF) | Replace with MEF or custom plugin system |
| XBAP (XAML Browser Applications) | Dead technology; use WASM target instead |
| `DDE` (Dynamic Data Exchange) | Replace with IPC, named pipes, or REST |
| COM Interop (in shared code) | Isolate to Windows head only with `#if WINDOWS` |

## 10. Windows-Only APIs

APIs that only work on the Windows head (`#if WINDOWS`) or must be abstracted:

### File System

| Windows API | Cross-Platform Strategy |
|-------------|------------------------|
| `System.IO.*` | Ō£ģ Works everywhere ŌĆö prefer this |
| `FileOpenPicker`, `FileSavePicker` | Ō£ģ Uno provides cross-platform wrappers |
| `StorageFile`, `StorageFolder` | Ō£ģ Uno supports |
| Win32 file dialogs (`IFileOpenDialog`) | `#if WINDOWS` only |
| `KnownFolders.*` | Partial; avoid on non-Windows |

### System / Shell

| API | Notes |
|-----|-------|
| `Process.Start("explorer.exe", ...)` | Windows only; use `Launcher.LaunchUriAsync` |
| `Registry` | Windows only; use `AppSettings` / `Preferences` |
| `Environment.GetFolderPath(SpecialFolder.*)` | Works on Desktop; not on mobile/WASM |
| `System.Drawing.Printing` | Windows only; use platform printing APIs |
| `WindowsIdentity` / `WindowsPrincipal` | Windows only; use platform auth |
| `Clipboard` (Win32) | Use `Windows.ApplicationModel.DataTransfer.Clipboard` |
| `Mutex` (global named) | Works on Desktop; not WASM |
| `WMI` (`System.Management`) | Windows only ŌĆö wrap with `#if WINDOWS` |
| Named pipes | Windows + Desktop Linux/Mac; not WASM |
| `TaskbarItemInfo` (WPF) | Windows only; no Uno equivalent |
| Jump Lists | Windows only via WinUI APIs |

### Abstracting Windows-only features

Use Uno's platform service pattern:
```csharp
// Define interface in shared project
public interface IPlatformService {
    Task<string?> PickFileAsync();
}

// Windows implementation
#if WINDOWS
public class WindowsPlatformService : IPlatformService {
    public async Task<string?> PickFileAsync() {
        var picker = new FileOpenPicker();
        // ...
    }
}
#endif

// Register in DI
services.AddSingleton<IPlatformService, WindowsPlatformService>();
```

Or use Uno.Extensions `IStoragePickerService` which abstracts this automatically.

## 11. Theming

### WPF theming model

WPF uses `ResourceDictionary` with `MergedDictionaries`, custom `ControlTemplate`s, and the
Windows Aero/Classic theme system. Themes are applied via `Application.Resources`.

### Uno Platform theming model

Uno uses the **WinUI 3 theming system** ŌĆö Fluent Design by default:

```xml
<!-- App.xaml -->
<Application.Resources>
    <ResourceDictionary>
        <ResourceDictionary.MergedDictionaries>
            <XamlControlsResources xmlns="using:Microsoft.UI.Xaml.Controls" />
            <ResourceDictionary Source="ms-appx:///MyApp/Themes/Colors.xaml" />
        </ResourceDictionary.MergedDictionaries>

        <!-- Override Fluent tokens -->
        <SolidColorBrush x:Key="SystemAccentColor" Color="#FF0078D4" />
    </ResourceDictionary>
</Application.Resources>
```

### Dark / Light mode

```xml
<!-- Automatic via RequestedTheme -->
<Application RequestedTheme="Dark" />
```

Or programmatically:
```csharp
((FrameworkElement)Window.Content).RequestedTheme = ElementTheme.Dark;
```

### Fluent Design tokens (replace WPF brushes)

| WPF Resource Key | Uno / WinUI 3 Fluent Token |
|-----------------|---------------------------|
| `SystemColors.WindowBrush` | `ApplicationPageBackgroundThemeBrush` |
| `SystemColors.WindowTextBrush` | `ApplicationForegroundThemeBrush` |
| `SystemColors.HighlightBrush` | `SystemAccentColorLight1` |
| `SystemColors.ControlBrush` | `ControlAltFillColorSecondaryBrush` |
| `SystemColors.GrayTextBrush` | `TextFillColorDisabledBrush` |

### Material Design / Custom themes

For non-Fluent looks, use:
- **Uno.Material** (`Uno.Material` NuGet) ŌĆö Google Material Design 3 theme
- **Uno.Cupertino** (`Uno.Cupertino` NuGet) ŌĆö Apple-style theme for iOS consistency

```csharp
// In App.xaml.cs
.UseMaterial()   // or .UseCupertino()
```

### WPF ControlTemplates ŌåÆ WinUI ControlTemplates

The templating system is the same concept, but template structures differ. Copy a WinUI 3 default
template from the [WinUI XAML Styles repo](https://github.com/microsoft/microsoft-ui-xaml) as a
starting point rather than adapting WPF templates directly.

## 12. Post-Migration UI Testing with the Uno Platform App MCP Server

Once the WPF app is migrated to Uno Platform, use the **Uno Platform App MCP Server** to drive the
running app and validate that UI, navigation, and interactive behaviour work correctly on each
platform target. This replaces the need to write separate UI test scripts manually ŌĆö the agent can
inspect and interact with the live app directly through MCP tools.

### What the Uno Platform App MCP Server provides

The Uno Platform App MCP Server is a local MCP server that exposes the running Uno app's UI tree
to AI agents. It supports two transports:

- **SSE (Server-Sent Events)** ŌĆö default; connects over HTTP to a running app
- **stdio** ŌĆö for CLI / pipeline use

When connected, the agent gains a set of MCP tools that map directly onto the live UI:

| MCP Tool | What it does |
|----------|-------------|
| `getVisualTree` | Returns the full XAML visual tree as structured JSON ŌĆö element types, names, properties, bounds |
| `getElement` | Fetches a single element by `AutomationId`, name, or type |
| `tapElement` | Simulates a tap / click on a named element |
| `typeText` | Types text into a focused or named TextBox / input |
| `pressKey` | Sends a keyboard key event (Tab, Enter, Escape, arrow keys, etc.) |
| `scrollElement` | Scrolls a ScrollViewer or list by amount or to a position |
| `getProperty` | Reads any property off a named element (Text, IsEnabled, Visibility, etc.) |
| `setProperty` | Sets a writeable property for test setup (e.g. force a state) |
| `navigateTo` | Triggers navigation to a named route or page type |
| `screenshot` | Captures a screenshot of the current UI state |
| `waitForElement` | Polls until an element appears, becomes visible, or reaches a property value |
| `getAppState` | Returns app-level state (current page, nav stack depth, window size) |

### Enabling MCP in the Uno app

Add the MCP server package to the shared project or each head:

```xml
<!-- In MyApp.csproj or each head .csproj -->
<PackageReference Include="Uno.Platform.McpServer" Version="*" />
```

Then enable it in `App.xaml.cs`:

```csharp
protected override void OnLaunched(LaunchActivatedEventArgs args) {
#if DEBUG
    this.EnableMcpServer(); // starts SSE listener on http://localhost:5000/mcp
#endif
    // ... rest of launch
}
```

The MCP server only runs in debug builds. Never ship it in release.

### Connecting an AI agent to the running app

Start the Uno app in debug mode (any head), then connect the agent:

```json
// mcp.json or Claude Desktop config
{
  "mcpServers": {
    "uno-app": {
      "type": "sse",
      "url": "http://localhost:5000/mcp"
    }
  }
}
```

Once connected, the agent should call `getAppState` first to confirm connection and identify the
current page before proceeding with any test steps.

### AutomationId ŌĆö the key to reliable element targeting

All interactive elements that agents need to target **must have an `AutomationId`** set. Add these
during migration (they are also best practice for accessibility):

```xml
<!-- Required for MCP targeting -->
<Button x:Name="SaveButton"
        AutomationProperties.AutomationId="SaveButton"
        Content="Save"
        Command="{x:Bind ViewModel.SaveCommand}" />

<TextBox AutomationProperties.AutomationId="UsernameInput"
         Text="{x:Bind ViewModel.UserName, Mode=TwoWay}" />

<Frame AutomationProperties.AutomationId="MainFrame" />
```

Naming convention: `PascalCase`, descriptive, unique per page. Do not reuse IDs across pages.

### Testing workflow for agents

After migration, follow this sequence for each platform head:

#### 1. Smoke test ŌĆö verify the app launches and reaches the expected first page

```
ŌåÆ getAppState
   assert: currentPage == "HomePage" or first expected route
ŌåÆ screenshot
   assert: visual tree is not blank / error state
```

#### 2. Button and command testing

```
ŌåÆ getElement(automationId: "SaveButton")
   assert: IsEnabled == true, Visibility == Visible
ŌåÆ typeText(automationId: "UsernameInput", text: "TestUser")
ŌåÆ tapElement(automationId: "SaveButton")
ŌåÆ waitForElement(automationId: "SuccessBanner", timeout: 3000)
   assert: element appears after save
```

#### 3. Navigation testing

```
ŌåÆ tapElement(automationId: "SettingsNavItem")
ŌåÆ waitForElement(automationId: "SettingsPage", timeout: 2000)
ŌåÆ getAppState
   assert: currentPage == "SettingsPage", canGoBack == true
ŌåÆ pressKey(key: "BrowserBack")   // or tapElement on back button
ŌåÆ getAppState
   assert: currentPage == "HomePage"
```

#### 4. Form validation testing

```
ŌåÆ tapElement(automationId: "SubmitButton")          // submit empty form
ŌåÆ waitForElement(automationId: "ValidationError")
ŌåÆ getProperty(automationId: "ValidationError", property: "Text")
   assert: contains expected error message
ŌåÆ typeText(automationId: "EmailInput", text: "invalid-email")
ŌåÆ tapElement(automationId: "SubmitButton")
ŌåÆ getProperty(automationId: "ValidationError", property: "Text")
   assert: contains email validation message
```

#### 5. List and data testing

```
ŌåÆ getVisualTree()
   find: ListView with AutomationId "ItemsList"
   assert: child count matches expected item count
ŌåÆ tapElement(automationId: "ItemsList_Item_0")       // tap first item
ŌåÆ waitForElement(automationId: "DetailPage")
ŌåÆ getProperty(automationId: "ItemTitle", property: "Text")
   assert: matches first item's expected title
```

#### 6. Theme and visual state testing

```
ŌåÆ tapElement(automationId: "ThemeToggle")
ŌåÆ getProperty(automationId: "AppShell", property: "RequestedTheme")
   assert: theme flipped between Light and Dark
ŌåÆ screenshot
   assert: visual appearance matches expected theme
```

### Cross-platform testing strategy

Run the same test sequence against each platform head in order. Track which heads pass or fail each
scenario:

| Test scenario | Windows | Skia Desktop | Android | iOS | WASM |
|---------------|---------|--------------|---------|-----|------|
| App launches  | Ō£ģ | Ō£ģ | Ō£ģ | Ō£ģ | Ō£ģ |
| Button tap    | ŌĆ” | ŌĆ” | ŌĆ” | ŌĆ” | ŌĆ” |

### Common migration issues found via MCP testing

| Symptom observed via MCP | Likely cause | Fix |
|--------------------------|-------------|-----|
| `getElement` returns null for a button | `AutomationId` missing | Add `AutomationProperties.AutomationId` |
| `tapElement` does nothing | Command not bound / `CanExecute` returns false | Check `{x:Bind ViewModel.Cmd}` and CanExecute logic |
| `waitForElement` times out after navigation | Page not registered in nav system | Verify route registration in Uno.Extensions nav |
| `getProperty("Text")` returns empty | Binding not firing ŌĆö `Mode=OneWay` missing | Add `Mode=OneWay` to `{x:Bind}` |
| Screenshot shows blank/white page | `XamlControlsResources` missing in App.xaml | Add as first `MergedDictionary` entry |
| Visual tree depth very shallow | Page content inside a `Frame` not yet navigated | Call `navigateTo` before inspecting content |
| `typeText` types but value doesn't update VM | TwoWay binding missing | Change `{x:Bind}` to `Mode=TwoWay` |
| Element visible on Windows but not Skia | Platform-conditional visibility | Check `#if` guards and `mc:Ignorable` prefixes |

### Accessibility and AutomationId audit

Before considering migration complete, run an audit:

```
ŌåÆ getVisualTree()
   for each Button, TextBox, ComboBox, ListView, NavigationViewItem:
     assert: AutomationProperties.AutomationId is set and non-empty
```

Flag any interactive element missing an `AutomationId` ŌĆö it cannot be reliably targeted by agents
or accessibility tools.
